# 我如何整理 TP-Link AX3000 的 Wi-Fi 設定：Home、IoT、Guest 分流，比追求「神設定」更重要

家用 Wi-Fi 設定最容易掉進一個陷阱：一直找「最佳參數」。

2.4 GHz 要不要固定頻道？5 GHz 要不要 160 MHz？SSID 要不要隱藏？Smart Connect 要不要開？WPA3 是不是越新越好？

這些問題都有技術答案，但真正讓網路穩定的，通常不是某個神奇開關，而是先把使用者分清楚。

我把 TP-Link AX3000 的設定整理成三個區域：

```text
Home  network: 筆電、手機、工作站、NAS
IoT   network: 智慧家電、感測器、家庭自動化設備
Guest network: 訪客，只能上網
```

這樣做的目的不是把家裡弄得像企業機房，而是讓日常使用更容易理解，也更容易除錯。

## 先決定誰應該碰到內網

很多人設定 Wi-Fi 時，第一個動作是改 SSID 名稱或調頻道。

我覺得更好的第一個問題是：

> 這台裝置需要看到我的內網資源嗎？

如果答案是「需要」，例如筆電要連 NAS、工作站要跑本機服務、手機要控制內網設備，那它屬於 Home network。

如果答案是「不一定」，例如智慧燈泡、冷氣、掃地機、攝影機、感測器，那它比較適合放在 IoT network。

如果答案是「絕對不需要」，例如親友來家裡借 Wi-Fi，那它應該放在 Guest network，而且最好只能上網，不能存取私人 LAN。

這個分類比單純追求速度重要。速度是體感，邊界是安全。

## 安全性：WPA3 很好，但相容性也要顧

Home network 可以優先使用：

```text
WPA3-Personal
```

如果所有主要裝置都夠新，這是乾淨的選擇。

但 IoT 裝置是另一個世界。很多智慧家電、感測器或老一點的手機，對 WPA3 支援並不完整。這時比較務實的設定是：

```text
WPA3-Personal + WPA2-PSK(AES)
```

這不是「比較不進步」，而是承認家用網路裡有各種年代的設備。與其讓 IoT 裝置一直連不上，不如把它們放在獨立區域，用相容性較好的安全模式，再透過隔離降低風險。

Guest network 也可以使用同樣的混合模式，重點是不要讓訪客進入私人 LAN。

## 不要把隱藏 SSID 當成安全功能

隱藏 SSID 聽起來很安全，實際上常常只是讓自己更難連。

SSID 被隱藏後，裝置仍然可能在連線過程中洩漏它正在尋找的網路名稱。對真正懂得掃描無線封包的人來說，它不是可靠的防護；對一般家用者來說，它反而可能造成手機、筆電、IoT 裝置重新連線不穩。

更值得做的是：

- 使用足夠長、不可猜的 Wi-Fi 密碼
- 關閉 WPS
- 使用 WPA2-AES 或 WPA3
- 將訪客網路與私人 LAN 隔離
- 不公開真實 SSID、QR code、路由器截圖

安全不是把名字藏起來，而是讓不該進來的人進不來。

## 2.4 GHz：不要嫌它慢，它是 IoT 的長腿

2.4 GHz 的優點不是快，而是穿牆與相容性。

建議起點：

```text
Channel width: 20/40 MHz Auto
Channel: Auto
Mode: 802.11b/g/n/ax
```

如果附近 Wi-Fi 很擁擠，可以手動測試三個經典頻道：

```text
1 / 6 / 11
```

對 IoT 來說，2.4 GHz 常常比 5 GHz 實用。智慧家電通常不需要大頻寬，它需要的是穩定地待在線上。把 IoT 固定放在 2.4 GHz，很多時候比讓它跟著 Smart Connect 漂來漂去更好理解。

## 5 GHz：留給真正需要速度的裝置

5 GHz 適合筆電、手機、工作站、串流與大型檔案傳輸。

建議起點：

```text
Channel width: 80 MHz
Channel: Auto
Mode: 802.11a/n/ac/ax
```

如果需要手動選頻道，可以從：

```text
36 / 149
```

開始測試。

5 GHz 的重點是把頻寬留給真正吃頻寬的裝置。NAS 備份、AI 工作站、遠端桌面、影音串流，這些比智慧插座更需要低延遲與高吞吐。

## OFDMA 和 TWT：Wi-Fi 6 的優點可以打開

AX3000 是 Wi-Fi 6 等級設備，OFDMA 和 TWT 通常值得開啟。

OFDMA 的價值在多裝置環境。它讓路由器更有效率地服務多台設備，而不是一台一台排隊。

TWT 則對省電有幫助，尤其是手機與 IoT 裝置。它可以讓設備更聰明地安排喚醒時間，減少不必要的無線活動。

建議：

```text
OFDMA: On
TWT: On
```

如果某台非常舊的設備出現相容性問題，再針對那台設備回頭調整。

## Smart Connect：方便，但不是每個環境都適合

Smart Connect 的想法很好：讓路由器自動替裝置選 2.4 GHz 或 5 GHz。

如果你只想讓家人輸入一組 Wi-Fi 名稱，然後什麼都不用管，Smart Connect 很方便。

但如果你有這些需求，它可能不適合：

- IoT 想固定在 2.4 GHz
- 工作站想固定在 5 GHz
- 你想快速判斷問題出在哪個頻段
- 你有 NAS、Home Assistant、Proxmox 或本機服務
- 你希望訪客、IoT、主要設備有清楚邊界

這種情況下，我會關閉 Smart Connect，改成明確分開 SSID：

```text
Home-2G
Home-5G
IoT
Guest
```

名字只是示意，公開文章中不應該放真實 SSID。

## 我最後整理出的設定表

| 項目 | 建議 |
| --- | --- |
| Home | WPA3-Personal；不相容時改 WPA3 + WPA2-AES |
| IoT | WPA3 + WPA2-AES |
| Guest | WPA3 + WPA2-AES，且與私人 LAN 隔離 |
| SSID | 顯示，不隱藏 |
| 2.4 GHz | 20/40 MHz Auto，Auto channel，必要時測 1/6/11 |
| 5 GHz | 80 MHz，Auto channel，必要時測 36/149 |
| OFDMA | 開啟 |
| TWT | 開啟 |
| Smart Connect | 需要明確分流時關閉 |
| WPS | 關閉 |

## 最重要的不是完美，而是可理解

家用 Wi-Fi 很少有永遠正確的設定。

隔壁鄰居換路由器，頻道就變了。家裡多一台 IoT，2.4 GHz 就更吵了。某台舊設備不支援 WPA3，你就得在安全性與相容性之間取平衡。

所以我不把這份設定看成「最終答案」，而是看成一個容易維護的起點：

1. 先分 Home、IoT、Guest。
2. 主要裝置盡量用 WPA3。
3. IoT 用相容模式，但放在獨立區域。
4. 不隱藏 SSID，不開 WPS。
5. 需要穩定分流時，關閉 Smart Connect。
6. 有問題時，再從頻道、頻寬、裝置相容性逐項排查。

好網路不是從神設定開始，而是從清楚的邊界開始。
